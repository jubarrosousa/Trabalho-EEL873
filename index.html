Apresentação de Projeto: Greet&Go

Autor

Pedro Ripper

Descrição informal

O conceito fundamental de Greet&Go é ser um aplicativo móvel que facilite e incentive a prática de esportes coletivos por meio da criação de comunidades. Nelas haveriam ferramentas para a organização e comunicação de locais, grupos e horários de prática das modalidades esportivas que elas representam. Além disso, haveriam também recursos inspirados nas redes sociais, como a criação de regras da comunidade, para aumentar o sentimento de pertencimento dos participantes.

Por exemplo: Alguém que gostaria de começar a praticar esgrima, mas que não sabe onde ou com quem aprender, poderia entrar no aplicativo e pesquisar por comunidades de praticantes de esgrima em locais próximos que aceitem iniciantes. Assim, ela poderia ir até o local no horário combinado e se juntar ao grupo para conhecê-los.

"Você gostaria de praticar uma certa atividade esportiva, mas não sabe onde, quando ou com quem? Isso nunca foi tão fácil quanto agora! Entre no app e veja se já existem comunidades do seu interesse, e então é só Greet&Go!"

Valor Entregue

Buscamos promover o sentimento de pertencimento na vida de cada usuário, por meio do estabelecimento de conexões com outros indivíduos em comunidades. Dentro dessas comunidades, buscamos organizar e simplificar a integração dos participantes, por meio das diversas ferramentas disponíveis.

Caso de uso

Exemplo: Atleta amador faz login, entra em uma nova comunidade esportiva e confirma presença em um evento dela.

Ator principal: Atleta amador

Pré-condições: Possuir uma conta no aplicativo

Cenário Principal:

1. Usuário faz login no sistema

  1.1. Sistema verifica as credenciais do usuário

  1.2. Sistema concede acesso ao aplicativo

2. Usuário acessa o mapa interativo

  2.1. Sistema verifica a localização do usuário

  2.2. Sistema mostra o mapa interativo na área do usuário

3. Usuário seleciona os filtros de seu interesse e pesquisa por uma comunidade esportiva

  3.1. Sistema verifica se existem comunidades que atendem aos filtros escolhidos

  3.2. Sistema mostra as comunidades mais próximas que atendem aos filtros escolhidos

4. Usuário escolhe uma comunidade e pede permissão para entrar nela

  4.1. Sistema verifica se o usuário atende aos critérios para entrar na comunidade escolhida

  4.2. Sistema envia a solicitação de entrada para os administradores da comunidade

  4.3. Sistema notifica que o usuário foi aceito e o adiciona à comunidade

  4.4. Sistema publica a informação

5. Usuário entra na página principal da comunidade e acessa a aba de eventos

  5.1. Sistema verifica se existem eventos futuros da comunidade

  5.2. Sistema mostra os eventos futuros da comunidade

6. Usuário escolhe um evento e confirma presença

  6.1. Sistema publica a informação e notifica o usuário sobre atualizações do evento

Cenários Alternativos:

1.1a. Credenciais incorretas

  1.1a.1. Sistema informa que as credenciais do usuário estão incorretas

  1.1a.2. Sistema pede para o usuário tentar novamente

  1.1a.3. Retorna ao caso 1

1.1b. Credenciais incorretas mais de uma vez

  1.1b.1. Sistema informa que as credenciais do usuário estão incorretas

  1.1b.2. Sistema sugere uma redefinição de senha

  1.1b.3. Usuário redefine (ou não) a senha

  1.1b.4. Retorna ao caso 1

2.1a. Sem acesso ao GPS

  2.1a.1. Sistema informa que não tem acesso ao GPS do usuário

  2.1a.2. Sistema pede para o usuário digitar um endereço

  2.1a.3. Usuário digita um endereço (ou liga o GPS)

3.1a. Sem comunidades que atendem aos critérios escolhidos

  3.1a.1. Sistema informa que não há comunidades que atendem perfeitamente aos critérios

  3.1a.2. Sistema mostra as comunidades com os critérios mais próximos dos escolhidos

4.1a. Usuário não atende aos critérios para entrar na comunidade

  4.1a.1. Sistema informa que o usuário não atende aos critérios da comunidade

  4.1a.2. Sistema pede que o usuário escolha outra comunidade

  4.1a.3. Retorna ao caso 4

4.2a. Administradores recusaram a solicitação de entrada do usuário

  4.2a.1. Sistema informa que a solicitação de entrada do usuário não foi aceita

  4.2a.2. Sistema pede que o usuário escolha outra comunidade

  4.2a.3. Retorna ao caso 4

5.1a. Sem eventos futuros

  5.1a.1. Sistema informa que não há eventos futuros marcados na comunidade

  5.1a.2. Sistema informa que irá notificar o usuário caso algum evento seja marcado
